# Roadmap

This roadmap is a high-level direction for my learning, projects, and portfolio.

## Phase 1 — Public portfolio foundation

Goal: Make my GitHub profile explain what I build clearly and honestly.

- Improve README
- Add project index
- Add case studies
- Add builder principles
- Add current focus and roadmap
- Keep private code private while documenting the ideas and learning

## Phase 2 — Better project documentation

Goal: Make every important project easier to understand.

Planned improvements:

- Add architecture notes for selected projects
- Add screenshots where safe
- Add short demo descriptions
- Add problem/solution/learning format to each case study
- Link public repositories where relevant

## Phase 3 — Practical AI demos

Goal: Create small public demos that prove practical AI skills without exposing private business systems.

Possible demos:

- Lead follow-up assistant demo with fake data
- Calendar-to-task-list demo with sample events
- AI workflow planner demo
- Small agent workflow prototype
- AI operations dashboard mock

## Phase 4 — Stronger engineering proof

Goal: Improve technical credibility.

Planned areas:

- More TypeScript projects
- Better tests
- Cleaner architecture docs
- API examples
- Deployment examples
- Small public utilities

## Phase 5 — Job-ready profile

Goal: Make the profile useful for applications, interviews, and student roles.

Focus:

- Clear project descriptions
- Honest skill level
- Strong portfolio structure
- Practical business examples
- Evidence of learning and initiative

## Long-term direction

I want to become strong at building useful AI-powered tools for real businesses.

The direction is not only coding. It is combining:

- Product thinking
- AI workflows
- Full-stack development
- Automation
- Business operations
- Technical systems
- Clear documentation
