# Case Study: Project Thea

## Summary

Project Thea is a Danish health-assistant prototype focused on helping users structure health concerns, prepare questions, and create clear summaries.

The project is not about replacing doctors or giving diagnoses. The focus is on communication, preparation, structure, and user guidance.

---

## Problem

Health-related information can be difficult for people to organize.

A user may have symptoms, worries, previous events, questions, and notes spread across memory, messages, and documents. That can make it hard to explain the situation clearly.

---

## Idea

Thea explores how AI can help users prepare better by turning messy information into structured summaries.

The assistant should help with:

- Organizing concerns
- Preparing questions
- Creating timelines
- Making short summaries
- Helping the user communicate more clearly
- Keeping clear safety boundaries

## What I learned

This project taught me that sensitive AI products require clear limits and careful design.

AI can be useful for organizing information, but the product must be honest about what it can and cannot do. Trust, safety, and clarity matter as much as the technical solution.
